# ADR 0008: Contact Profile 触碰属性反馈

**日期**: 2026-07-11
**状态**: Accepted

---

## 背景

ADR 0007 让手触记忆和杖触记忆共享触觉记忆系统：二者都能在世界空间留下局部显影点，杖触成功生成记忆点时播放声音。

下一步需要让不同对象或部件被触碰时产生不同反馈。例如石质路面、柏油路面、玻璃门、金属柱等应显示不同颜色；盲杖触碰时还应发出不同声音。这里的分类不能直接叫“材质”，因为声音不完全由真实材质决定：钢柱和钢管可能同为金属，但敲击声音不同。Godot 的渲染 `Material` 也不应承担触碰反馈语义。

## 决策

引入 **触碰属性 (Contact Profile)** 作为对象或部件被手触/杖触感知时使用的反馈配置。它不表示真实材质、渲染材质或物体身份，只表示触碰反馈。

实现方向采用 `ContactProfile` Resource 和 `ContactProfileProvider` 组件，而不是节点 metadata、散落字典或 Godot `Material`：

- `ContactProfile` 保存触碰反馈数据，例如 `id`、`display_name`、`reveal_color`、`cane_sound_id`。
- `ContactProfileProvider` 挂在可触碰对象或部件下，引用一个 `ContactProfile`。
- 命中节点解析触碰属性时，从命中节点开始查找最近的 provider；找不到就逐级向父节点回溯；仍找不到则使用默认触碰属性。

复合物体使用局部覆盖规则：部件上的局部触碰属性优先于整体对象上的触碰属性。一个节点最多拥有一个有效触碰属性，避免同一触碰点产生歧义。

显影颜色与声音独立配置。`reveal_color` 同时用于手触和杖触，避免同一对象因触碰来源不同而增加额外学习成本。`cane_sound_id` 只用于杖触；手触永远不播放声音。

杖触声音仍遵守 ADR 0007 的约束：只有成功生成杖触记忆点时才播放声音。`cane_sound_id` 允许为空，表示杖触只显影不发声。若配置了不存在的 sound id，调试模式应报警并静默，不回退到默认声音，避免用错误声音误导玩家。只有完全缺失触碰属性时才使用默认触碰属性。

触觉记忆点必须独立保存自己的显影颜色。后续触碰到其他触碰属性时，旧记忆点不能一起变色。多个不同颜色记忆点重叠时，MVP 采用“当前像素贡献最强的记忆点颜色”；强度相同则偏向较新的记忆点，不做颜色混合。

接触节流需要纳入触碰属性变化：同一位置持续接触同一触碰属性时仍按距离和时间节流；但一旦触碰属性变化，即使距离和时间没有达到阈值，也视为新的触觉事件，生成新记忆点并触发对应杖触声音。

`ContactProfile` 不控制触觉记忆点的半径、寿命或衰减。生命周期仍由触碰来源决定：手触记忆使用手触参数，杖触记忆使用杖触参数。触碰属性只控制显影颜色和杖触声音。

## MVP 边界

MVP 只实现：

- 对象/部件级触碰属性配置。
- 父级回溯和默认触碰属性兜底。
- 每个触觉记忆点独立颜色。
- 杖触按触碰属性播放声音，手触不播放声音。
- 触碰属性变化绕过接触节流。
- 调试日志暴露 profile 解析、fallback、节流、声音触发情况。

MVP 暂不实现：

- 根据碰撞力度、速度、角度改变声音。
- 连续摩擦声。
- 从渲染材质或物理材质自动推断触碰属性。
- 复杂混音、随机音频池或声学传播模拟。
- 手触和杖触使用不同显影颜色。
- 编辑器批量校验工具。

## 验收要求

验收不能只启动游戏确认无报错。需要在 `scripts/debug/` 中添加临时测试脚本，控制角色或直接驱动对应系统触发以下场景，并通过 Godot 输出确认结果：

- 杖触不同 profile 的对象，生成对应颜色并播放对应声音。
- 手触同一对象，只生成对应颜色，不播放声音。
- 复合物体相邻部件具有不同 profile 时，profile 变化绕过节流并生成新记忆点。
- 未配置对象使用默认触碰属性，并在 debug 输出 fallback 警告。
- 同一 profile 同一点持续接触不刷屏。

临时测试脚本完成后必须移除。

## 考虑的替代方案

| 方案 | 否决理由 |
|------|---------|
| 直接使用 Godot `Material` 或真实材质名 | 触碰反馈不是渲染材质；声音也不完全由真实材质决定 |
| 节点 metadata 存 profile id | 实现很轻，但属于隐式协议，缺少类型约束和编辑器可发现性 |
| 在每个节点上散落 `{name, color, sound_id}` 字典 | 重复配置多，字段易拼错，后续统一调整困难 |
| 颜色和声音绑定成单一材质枚举 | 无法表达同色不同声、同材质不同结构的对象 |
| profile 控制记忆点半径和寿命 | 会混淆“触碰属性”和“触碰来源”，增加调参复杂度 |
| 不让 profile 变化绕过节流 | 玻璃门板和金属把手等相邻部件可能被节流吞掉，玩家无法感知差异 |
