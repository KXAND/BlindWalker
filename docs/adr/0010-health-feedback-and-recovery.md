# ADR 0010: Health Feedback and Recovery

**日期**: 2026-07-13
**状态**: Accepted

---

## 背景

玩家受到伤害后，目前只有 HP 数字和音效反馈。低血量状态缺少持续压迫感，也没有静止休整后的恢复机制。项目需要让受伤、濒死和休整回血形成一个清晰的闭环。

## 决策

引入两类视觉反馈：

- **受伤闪光**：每次 `player_damaged` 触发一次短促红色边缘闪光。
- **濒死红光**：当 HP 小于等于最大 HP 的 25% 时，屏幕边缘持续闪烁红光。

濒死红光随 HP 比例变化，HP 越低，红色越向屏幕内部延伸。最大侵入范围限制为屏幕半径的 25%，避免遮挡中心核心视野。视觉强度和范围需要平滑追随 HP，不能因回血或受伤出现明显跳变。

受伤闪光优先于濒死红光，但只持续短时间。两者叠加时取更强效果，不做简单相加，避免画面过曝。

## 静止回血

静止按“移动输入意图”判定，而不是按实际位置变化判定。玩家被墙卡住但仍按住移动键时，不算静止。

回血规则：

- 玩家无移动输入持续 3 秒后开始回血。
- 移动或受到伤害会重置 3 秒等待。
- HP 为 0 时不回血，仍进入失败流程。
- 回血速度按当前 HP 比例非线性衰减：低血量回复快，高血量回复慢，但整体速度保持克制，避免静止数秒就快速回满。
- 本轮不新增回血音效或额外 UI，只通过 HP 数字和红光退去反馈。

回血公式：

```text
heal_per_second = lerp(0.5, 5.0, 1 - hp_ratio)
```

## 实现边界

- `PlayerAttributes.gd` 负责 HP 数值、回血计时和非线性回血。
- `GaitController.gd` 暴露移动意图状态，供回血逻辑查询。
- `HealthUI.gd` 负责 HP 数字、受伤闪光和濒死红光，不新增专门 UI 脚本。

## 非目标

本轮不实现：

- 0 HP 复活。
- 回血音效。
- 新的回血进度条。
- 与剧情/对话系统联动的回血规则。
