# ADR 0009: Narrative Sequence 顺序剧情演出

**日期**: 2026-07-13
**状态**: Accepted

---

## 背景

现有 `CutsceneManager` 只支持按 `cutscene_id` 显示单句硬编码字幕，并附带简单相机位移动画。项目后续需要表达更完整的剧情，例如小林和李奶奶之间的多句对话、旁白，以及未来可能接入的语音。

该功能不应被设计成完整分支对话系统。当前目标是顺序播放文本和音频，支持说话人和旁白，能在剧情期间保护玩家状态，并允许玩家跳过当前单句或整个序列。

## 决策

引入 **叙事序列 (Narrative Sequence)** 作为线性剧情数据模型。一个叙事序列由多条 **叙事行 (Narrative Line)** 按数组顺序组成。

`CutsceneManager` 继续作为播放入口，不在本轮重命名或拆成新系统。它从“播放单条硬编码字幕”升级为“播放叙事序列”。这样能复用已有字幕、输入闸门和 `cutscene_started` / `cutscene_ended` 事件，同时避免不必要重构。

叙事序列至少包含：

- `sequence_id`
- `lines`
- `lock_input`
- `lock_gameplay`
- `default_line_duration`

叙事行至少包含：

- `speaker_id`
- `speaker_name`
- `text`
- `audio_id`
- `duration`

`speaker_id` 用于逻辑和未来样式扩展，`speaker_name` 用于 UI 显示。旁白允许没有说话人；旁白播放时不显示空 speaker 区域。

## 播放规则

台词顺序只由 `lines` 数组顺序决定，不引入 `next_id`、分支或流程图。

每条叙事行按显式 `duration` 自动推进。若存在 `audio_id`，播放时调用 2D 音频接口；音频缺失不阻塞剧情，调试模式应输出缺失信息，并继续按文本时长推进。内容制作时应保证 `duration` 不短于对应音频长度。

播放期间叙事输入优先级高于 gameplay 输入。单句跳过和整段跳过是基础能力，不做成可选配置：

- `SPACE` 跳过当前叙事行。
- `ESC` 跳过整个叙事序列。

跳过当前叙事行时，当前叙事音频必须停止，然后进入下一行。跳过整个叙事序列时，必须隐藏 UI、停止当前叙事音频、恢复演出锁定状态，并发出演出结束事件。

## 演出锁定

`lock_input` 默认开启，用于禁用玩家移动、视角、盲杖、触摸等输入。

`lock_gameplay` 默认开启，用于避免剧情播放期间发生非预期玩法后果。它不只是“锁血”，还应覆盖伤害、摔倒、踉跄、失败、胜利等状态变化入口。实现时应确保结束、跳过或场景退出时恢复锁定状态，避免永久锁死输入或 gameplay。

## 非目标

本轮不实现：

- 分支对话。
- 玩家选项。
- 手动按键推进作为唯一推进方式。
- 镜头调度。
- 每行自定义 UI 样式。
- 从音频资源自动推导台词时长。
- 将现有 NPC 对话系统整体迁移为叙事序列。

未来 NPC 对话可以复用叙事行数据结构，但本轮不强行合并 `NpcDialogue`。

## 数据形式

叙事数据应使用 Godot `Resource` / `.tres`，而不是散落字典或 JSON。理由是 `.tres` 更符合 Godot 编辑器工作流，能在 Inspector 中配置，也更容易对字段做类型约束。

音频字段使用 `audio_id`，对应音频管理器中的注册 ID，不在叙事行中直接存文件路径。

## 验收要求

实现阶段不能只启动游戏确认无报错。需要在 `scripts/debug/` 添加临时测试脚本，自动触发一个叙事序列并跟踪 Godot 输出，至少确认：

- 按顺序播放多条叙事行。
- 人物台词显示说话人，旁白不显示说话人。
- 有音频的叙事行会请求对应 `audio_id`。
- 缺失音频不阻塞播放。
- `SPACE` 跳过当前叙事行。
- `ESC` 跳过整个叙事序列。
- 播放期间输入和 gameplay 被锁定，结束或跳过后恢复。
- 临时测试脚本完成后必须移除。

## 考虑的替代方案

| 方案 | 否决理由 |
|------|---------|
| 继续在 `CutsceneManager` 中按 `cutscene_id` 硬编码文本 | 多句剧情和音频会快速膨胀，难以维护和验收 |
| 新建完整 Dialogue 系统 | 当前不需要分支、选项和 NPC 状态机，过度设计 |
| 使用 JSON 存剧情 | 与 Godot 编辑器工作流不如 Resource / `.tres` 贴合，字段校验更弱 |
| 直接在叙事行中写音频文件路径 | 会绕过音频注册和降级逻辑，不利于后续统一管理 |
| 将跳过能力做成可选配置 | 会让剧情交互不一致，增加测试复杂度 |
| 本轮依赖音频实际长度推进 | 音频资产和导入状态不稳定时容易卡住剧情，显式 duration 更可控 |
