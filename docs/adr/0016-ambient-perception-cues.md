# ADR 0016: Ambient Perception Cues

**日期**: 2026-07-14
**状态**: Accepted

---

## 背景

我们希望在街道场景中加入太阳和风，但玩家是视障者，太阳不能被实现成真实可见天空或真实光照，风也不能被实现成物理风场。它们应表达角色对热源和气流的身体感受，而不是给玩家额外的空间视觉信息。

## 决策

新增“环境感知提示”作为独立感知层。热源提示用于表达太阳带来的模糊热感方向，气流提示用于表达玩家周围的风向、风强和流动感。

环境感知提示不使用真实光照系统，不改变 `WorldEnvironment` 的照明，不投影，不让物体变亮。热源提示可以采用天空盒式表现，但必须作为独立感知层，而不是真实天空盒；表现上应是低精度的模糊热感光晕，不是清晰太阳圆盘。

气流提示只在玩家周围自动显现，不做全场风场。它不碰撞、不绕物、不避障、不根据墙体、道路或门洞改变形态，也不揭示空间结构。本轮只在街道场景使用气流提示，不实现通用室内外区域系统。

环境感知提示不生成触觉记忆点，不参与互动提示的显影判断，不影响可互动对象是否可见。互动提示仍只依赖手触、杖触等明确感知来源。

环境感知提示属于低优先级世界感知层，视觉强度应低于手触/杖触显影，并低于 QTE、教程、叙事、交互提示和失败 UI。

## 考虑的替代方案

| 方案 | 否决理由 |
|------|---------|
| 使用真实 `DirectionalLight3D` 表达太阳 | 会照亮场景并改变玩家可见信息，破坏黑暗探索体验 |
| 使用真实天空盒 | 容易被理解成玩家看见天空或环境背景，并提供过强方向锚点 |
| 使用物理风或与场景交互的风场 | 会通过绕墙、避障、门洞流动等形态暴露空间结构 |
| 放入 `TouchMemorySystem` | 太阳和风不是接触事件，不应生成或管理触觉记忆点 |

## 验收要求

实现阶段不能只启动游戏确认无报错。需要在 `scripts/debug/` 添加临时测试脚本，控制玩家或相机朝不同方向观察，并跟踪 Godot 输出，至少确认：

- 热源提示不会改变真实光照、不会让物体变亮。
- 热源提示表现为模糊热感光晕，而不是清晰太阳圆盘。
- 气流提示只在玩家周围出现，并按配置方向流动。
- 气流提示不与墙体、道路、门洞或障碍物发生交互。
- 环境感知提示不生成触觉记忆点，也不让互动提示变为可见。
- QTE、教程、叙事、交互提示显示时优先级高于环境感知提示。
- 临时测试脚本完成后必须移除。
