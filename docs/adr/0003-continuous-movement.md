# ADR 0003: 连续移动取代离散步态

**日期**: 2026-07-09  
**状态**: Accepted  
**取代**: ADR-0001

---

## 上下文

ADR-0001 决定采用离散迈步（W=左脚、E=右脚交替）作为移动方式，核心理由是"每一步都是有意决策"。

在实际原型开发（`ref/prototype` 分支）中，发现以下问题：

1. **双键交替操作门槛高**：W/E 交替迈步 + 鼠标扫杖 + R 键视角 = 三组输入同时管理，认知负荷过大
2. **步态状态机复杂度溢出**：BothEven/LeftAhead/RightAhead/Locked 四状态 + 同脚锁定/解锁逻辑，实现易出 bug（如 stagger 中断时 locked_key 不清空导致永久锁死）
3. **离散步长与连续世界的体感割裂**：固定 0.5m 离散位移在连续 3D 空间中显得机械，与盲杖扫动的连续性不协调

## 决策

改为**单键连续移动**：按住 W = 沿视角前方持续行走，松开 = 停止。

### 具体设计

| 机制 | 设计 |
|------|------|
| 基础移动 | 按住 W = `WALK_SPEED`（~0.8 m/s）沿 `-basis.z` 前进，松开 = 停止 |
| 谨慎模式 | 按住 SHIFT + W = `CAUTIOUS_SPEED`（~0.3 m/s）减速行走，下楼梯不摔 |
| 高抬腿模式 | 按住 SPACE + W = `HIGH_STEP_SPEED`（~0.3 m/s）减速行走，允许跨过 ≤ `MAX_HIGH_STEP_HEIGHT`（0.3m）的台阶。**固定高度，非蓄力** |
| 撞墙 | `move_and_slide()` 碰撞响应 + 短暂减速 + `wall_hit` 音效，**不扣血** |
| 上楼梯 | 每帧射线检测前方地形高差：高差 > 0 且 ≤ 0.3m + SPACE 按住 → 平滑抬升 `global_position.y`；SPACE 未按住 → 撞墙处理 |
| 下楼梯 | 每帧射线检测前方地形高差：高差 < -0.15m + SHIFT 未按住 → 摔倒扣血 |
| 摔倒 | 扣 `FALL_DAMAGE`（20 HP）+ `fall` 音效 + `player_fell` 信号，短暂禁用移动（1-2s 恢复） |
| 脚步音频 | 距离累加器：每移动 `STEP_AUDIO_DISTANCE`（0.5m）播放一次脚步声，交替左右声道 |

### 废弃内容

- BothEven / LeftAhead / RightAhead / Locked 状态机
- 同脚锁定机制（G4-G6）
- `gait_state_changed` EventBus 信号
- `KEY_LEFT_FOOT` / `KEY_RIGHT_FOOT` 常量
- `HIGH_STEP_CHARGE_RATE` 蓄力机制
- `STEP_LENGTH_FLAT` / `STEP_LENGTH_STAIR` 离散步长

### 保留内容

- SHIFT 谨慎模式（G7）→ 改为减速行走
- SPACE 高抬腿模式（G8）→ 改为固定高度开关
- 撞墙踉跄不扣血（G9）
- 楼梯判定（G10-G11）→ 从每步检测迁移为每帧检测
- `MAX_HIGH_STEP_HEIGHT` 常量
- `FALL_DAMAGE` / `STAGGER_PUSH_BACK` 常量

## 核心挑战的转移

ADR-0001 的核心论点是"离散迈步让每一步成为有意决策"。连续移动模式下，挑战类型发生变化：

| 维度 | 离散步态（ADR-0001） | 连续移动（本 ADR） |
|------|---------------------|-------------------|
| 操作复杂度 | W/E 交替 + 锁定管理 | 单键按住 |
| 挑战来源 | 每步离散决策 | 持续注意力分配 + 速度自控 |
| 盲杖探测节奏 | 先探再走（步间停顿天然提供探测窗口） | 边走边探（需主动管理扫杖覆盖） |
| 摔倒惩罚 | 一步走错的明确后果 | 走太快没探测到的模糊后果 |
| 摔倒严重度 | 扣血 + 踉跄 | 扣血 + 1-2s 移动禁用（强制停下重新探测） |

**缓解措施**：移动速度默认较低（0.8 m/s = 慢步行走），玩家仍需频繁用盲杖扫探前方。摔倒后短暂禁用移动，强制玩家停下来重新评估。

## 替代方案

| 方案 | 否决理由 |
|------|---------|
| 保留离散步态但简化状态机 | 状态机与连续世界的体感割裂仍存在，且简化后仍比单键复杂 |
| 连续移动 + 离散步频音频 | 音频步频与实际位移解耦已通过距离累加器实现，不需要离散步态 |
| 连续移动 + 体力/疲劳系统 | MVP 范围外，过度设计 |

## 影响

- GaitController 从状态机驱动改为速度驱动，代码大幅简化
- InputManager 从 W/E 双键改为 W 单键持续检测
- EventBus 删除 `gait_state_changed` 信号
- GameConfig 常量更新（见 PRD §2.0）
- ADR-0001 标记为 Superseded
