# ADR 0011: Stumble, Fall, and Tumble Feedback

**日期**: 2026-07-14
**状态**: Accepted
**修正**: ADR-0003

---

## 背景

ADR-0003 将连续移动中的失败反馈定义为简单的踉跄、扣血和短暂禁用移动。实际体验中，这不足以表达“身体失去平衡”的过程：撞墙、台阶失败、下楼梯未谨慎、摔在楼梯或斜坡上，应该具有不同严重度和不同恢复路径。

本 ADR 修正 ADR-0003 中踉跄、摔倒和下楼梯失败的语义。

## 决策

### 踉跄分级

踉跄拆分为两级：

- **轻踉跄**：撞墙、小碰撞等轻微失衡。只产生短促镜头不稳、音效和短暂停顿；不扣血，不进入 QTE。
- **失衡踉跄**：未按 SPACE 处理上台阶、危险障碍、扶手保护下的下楼失败等严重失衡。进入挽回 QTE。

踉跄期间允许玩家继续移动，但移动会延长从踉跄恢复到稳态的过程。轻踉跄期间如果再次触发失衡踉跄，则升级为失衡踉跄。失衡踉跄期间如果再触发轻踉跄或失衡踉跄，则直接摔倒。

### 挽回 QTE

失衡踉跄提供一次挽回动作，默认窗口由失衡进度条表达：进入失衡时进度为 `0%`，若不挽回则在 `UNSTABLE_STUMBLE_QTE_WINDOW` 内增长到 `100%` 并摔倒。按住 `SHIFT + SPACE` 时，进度按 `UNSTABLE_STUMBLE_QTE_HOLD_TIME` 对应速度向 `0%` 回落；进度回到 `0%` 时恢复稳态。因此在 `50%` 失衡进度按住时，只需要约一半 `QTE_HOLD_TIME` 即可恢复。移动会降低恢复速度。

挽回输入为 `SHIFT + SPACE`。该输入在 QTE 判定期间优先级高于普通模式切换，因此不会同时被解释为谨慎模式或高抬腿模式。

- QTE 成功：恢复稳态。
- QTE 失败：进入摔倒。

### 摔倒与滚落

摔倒后不再提供 QTE，并锁定视角。玩家只能等待稳定平面允许起身。

玩家在低矮台阶、路牙等可高抬腿跨越的高度上失衡并最终摔倒时，摔倒过程会沿原移动方向越过该高度差，并落到上方平面；这等效于“失败地跨上去”，而不是被路牙原地卡住。

滚落不是独立入口状态，而是摔倒恢复过程中的结果：玩家摔倒后尝试起身；如果仍处于楼梯、斜坡或连续下坡等不稳定地形，则起身失败并继续滚动，直到到达稳定平面。

稳定平面的判断原则：

- 脚下坡度低于阈值。
- 前方短距离内没有继续下降的台阶或斜坡。
- 未来可由场景标记覆盖，例如扶手安全区或不稳定坡面。

到达稳定平面后进入固定起身时间，默认 `1.2s`。起身期间仍锁定视角；起身完成后恢复正常控制。

### 伤害结算

摔倒第一帧不结算伤害。滚落期间每 `1.0s` 结算一次伤害。每次摔倒/滚落至少结算一次伤害；滚落结束时，如果本次还没有结算过伤害，立即结算一次；如果已经结算过，且距上次结算少于 `0.7s`，不补结算；否则补结算一次。

单次摔倒/滚落累计伤害上限为 `80`。该上限只限制本次摔倒/滚落造成的累计伤害，不影响其他伤害来源。

如果 HP 在摔倒或滚落中归零，立即进入失败流程，不等待起身完成。

## 镜头调度

摔倒、滚落、起身和踉跄都必须通过镜头调度表达身体状态。镜头运动效果是本需求的核心验收对象，而不是可选的视觉增强：

- 轻踉跄：短促不稳定镜头反馈。
- 失衡踉跄：更明显的不稳定镜头反馈，并提示 QTE 风险。
- 摔倒：明显下坠、倾斜、贴近地面的镜头反馈。
- 滚落：持续失控的镜头反馈。
- 起身：镜头从低处和倾斜状态恢复到正常状态。

镜头调度只影响视觉表现，不改变玩家真实朝向，不改变盲杖、触摸或目标检测方向。摔倒、滚落和起身期间锁定视角。

镜头强度应保持克制，避免过强晃动造成不适。后续可以加入“降低镜头晃动”选项。

实现上应有独立的相机运动职责边界，用于播放踉跄、摔倒、滚落和起身的临时视觉偏移。该职责不应由输入解释层承担，也不应把玩家真实朝向、盲杖方向或触摸方向作为镜头效果的一部分一起改掉。

## 保留的输入语义

- `SHIFT` 仍是谨慎下楼的声明。
- `SPACE` 仍是高抬腿上台阶的声明。
- `SPACE` 不保护下楼梯。
- 撞墙仍不扣血。
- 未来可出现扶手等上下文状态，将下楼失败从摔倒降级为失衡踉跄。

## 非目标

本 ADR 不要求真实 ragdoll。滚落应是确定性的失控状态，而不是把玩家拆成物理刚体后交给 ragdoll 求解。

本 ADR 不规定相机运动、QTE 状态机或地形检测的具体类名和算法，但要求最终实现必须包含可验收的镜头运动效果、QTE 挽回流程和稳定平面判定。
